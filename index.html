// 01 — capabilities

Every layer of
your secrets lifecycle.

From generation to rotation to destruction — with cryptographic guarantees at every step.

[AES]
Envelope Encryption
Each secret gets its own DEK. DEKs are wrapped by a KEK derived via HKDF-SHA256 from a root key that never touches disk.
[SS]
Shamir's Secret Sharing
The root key is split into N shards. Any threshold of operators must cooperate to unseal. No single point of compromise.
[KV]
Versioned KV Store
Every write creates a new version. Soft-delete, undelete, or permanently destroy any version. Full history, full control.
[POL]
Path-Glob Policies
Fine-grained ACL using path glob patterns. secret/data/*, **, exact match — attach multiple policies to any token.
[AUTH]
Token + AppRole Auth
Human tokens for interactive use. AppRole (role ID + short-lived secret ID) for CI/CD and service-to-service auth.
[LOG]
Tamper-Evident Audit
Every request logged — token hash, path, status, latency. Secret values never written. Postgres trigger enforces append-only.
[PEM]
TLS / PEM Secrets
Structured storage for certificates with automatic expiry parsing. Know when your cert expires without decrypting it.
[ENV]
Env Bundles
Store environment variable sets as a first-class secret type. Export to .env format instantly for local dev or CI injection.
[MTR]
Prometheus Metrics
Request counts, latency histograms, active token count, secrets by type, and real-time seal status — all at /metrics.

// 03 — security properties

Defense in depth.

No single failure mode compromises the whole system.

[01]
Data Unreadable at Rest AES-256-GCM with per-secret DEKs. KEK held only in RAM. Full DB exfiltration yields only ciphertext.
[02]
No Single Key Holder Shamir's Secret Sharing: threshold of operators required. No one person can unseal the vault alone.
[03]
Tokens Never Stored Only SHA-256 hashes in the database. Plaintext shown once at creation. DB dump cannot be used to authenticate.
[04]
Append-Only Audit Postgres trigger rejects any UPDATE or DELETE on audit_log. Tamper-evident by design.
[05]
Least-Privilege Access Path-glob policies. Tokens limited to declared capabilities. No ambient authority.
[06]
Short-Lived Credentials AppRole secret IDs expire by TTL or use count. Tokens carry TTLs. Revocation cascades to children.
